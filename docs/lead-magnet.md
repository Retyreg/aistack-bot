# AIstack — Стартовый набор промптов (лид-магнит)

Это подарок, который бот отдаёт **после диагностики**. Три набора — по ветке результата. Каждый промпт самодостаточный: роль → задача → что подставить → формат ответа. Работает в ChatGPT, Claude, Gemini.

> Отдельный лид-магнит на **входе с вебинара** (PDF «Конкурентный анализ v2.0» + запись эфира, до диагностики) — см. `docs/funnel-texts.md`, раздел 0. Эти сегментные наборы остаются как есть и выдаются после диагностики.

**Как пользоваться:** скопируй промпт, замени всё в `[квадратных скобках]` на свои данные, вставь в чат с нейросетью. Меняй и комбинируй под себя — это и есть навык, который качаем на курсе.

# 🟢 Набор «AI-Маркетолог»

Для тех, кого мало кто видит. Эти 4 промпта закрывают контент на месяц вперёд.

## 1. Контент-план на месяц

```
Ты — мой AI-маркетолог. Составь контент-план на 4 недели для проекта.

О проекте:
— Что делаю: [продукт/услуга в 1 предложении]
— Для кого: [целевая аудитория, регион]
— Каналы: [Instagram / Telegram / TikTok — что ведёшь]
— Цель на месяц: [подписчики / заявки / продажи — что важнее]

Дай таблицу: неделя → день → формат (пост/рилс/сторис) → тема → цель поста (охват/вовлечение/продажа).
Темы — конкретные, под мою нишу, а не «расскажите о себе».
Чередуй пользу, личное и продающее в пропорции 60/20/20.
```

## 2. Пост из одной мысли

```
Ты — мой копирайтер. Разверни мою сырую мысль в готовый пост для [Telegram / Instagram].

Мысль: [пиши как есть, своими словами, можно коряво]
Аудитория: [кто читает]
Цель поста: [польза / прогрев / продажа]

Структура: цепляющий первый абзац (без «сегодня хочу рассказать»), тело с конкретикой и примером, финал с одним призывом.
Тон — живой, разговорный, без канцелярита и эмодзи через слово.
Длина — до 1200 знаков. Дай 2 варианта первого абзаца на выбор.
```

## 3. Серия рилсов/коротких видео

```
Ты — сценарист коротких видео. Придумай 5 идей рилсов под мой проект.

Проект: [что делаю, для кого]
Что хочу от видео: [узнаваемость / заявки]

Для каждого рилса дай:
— Хук (первые 3 секунды, текст на экране)
— О чём видео (3-4 пункта сценария)
— Призыв в конце
Идеи — разные по формату (история, разбор, ошибка, до/после, ответ на вопрос).
Под мою нишу, реалистичные для съёмки на телефон в одиночку.
```

## 4. Прогревающая рассылка

```
Ты — мой маркетолог. Напиши серию из 3 сообщений для рассылки по базе перед [запуском продукта / акцией].

Что продаю: [продукт, цена, для кого]
База: [кто эти люди, насколько тёплые]
Оффер: [что предлагаю, есть ли дедлайн/ограничение]

Сообщение 1 — польза + лёгкий намёк на тему.
Сообщение 2 — пруфы и снятие главного возражения.
Сообщение 3 — прямой оффер с дедлайном.
Каждое — до 600 знаков, с одним призывом. Тон живой, без давления.
```

# 🔵 Набор «AI-Операционник»

Для тех, кто тонет в рутине. Эти промпты разгружают переписку, заявки и отчёты.

## 1. Шаблоны ответов клиентам

```
Ты — мой ассистент поддержки. Собери базу быстрых ответов на частые вопросы клиентов.

О проекте: [что продаю, кому]
Частые вопросы: [перечисли 5-7, как их реально задают]

Для каждого вопроса дай готовый ответ: дружелюбный, по делу, без воды, с призывом к следующему шагу где уместно.
Тон — [официальный / на «ты», как у меня принято].
Добавь 2-3 ответа на сложные ситуации: задержка, недовольство, возврат.
```

## 2. Обработка входящей заявки

```
Ты — мой менеджер по заявкам. Я дам сырую заявку от клиента — разложи её и предложи следующий шаг.

Заявка: [вставь сообщение клиента как есть]
Что я продаю: [продукт, вилка цен]

Дай:
1) Что человек хочет (одним предложением)
2) Чего не хватает — какие 2-3 вопроса задать, чтобы понять задачу
3) Готовый текст ответа клиенту с этими вопросами
4) Тёплый/холодный лид и почему
```

## 3. Еженедельный отчёт за 5 минут

```
Ты — мой операционный помощник. Собери из моих заметок понятный недельный отчёт.

Мои заметки за неделю (как попало):
[вставь всё подряд — что сделал, цифры, проблемы]

Сделай отчёт:
— Главное за неделю (3 пункта)
— Цифры (если есть — вынеси отдельно)
— Что застопорилось и почему
— Фокус на следующую неделю (3 задачи)
Коротко, по-деловому, чтобы читалось за минуту.
```

## 4. Регламент рутинного процесса

```
Ты — операционный директор. Опиши пошаговый регламент для моего повторяющегося процесса, чтобы его можно было делегировать или автоматизировать.

Процесс: [например — «обработка заявки от первого сообщения до оплаты»]
Как делаю сейчас: [опиши шаги, как помнишь]
Инструменты: [что использую — Telegram, таблицы, CRM]

Дай:
1) Чёткие шаги по порядку (нумерованно)
2) Что на каждом шаге можно автоматизировать или поручить AI
3) Где сейчас теряется время или клиенты
Простым языком, без теории.
```

# 🟣 Набор «AI-Продакт»

Для тех, у кого есть идея/аудитория, но нечего продать. Эти промпты доводят до того, что можно показать клиенту.

## 1. Идея → описание продукта

```
Ты — продакт-менеджер. Помоги превратить мою идею в понятное описание продукта.

Идея: [опиши своими словами]
Для кого: [аудитория, регион]
Какую боль решает: [проблема клиента]

Дай:
— Продукт в 1 предложении (так, чтобы понял любой)
— Кому и зачем (3 сегмента + их выгода)
— Минимальная версия (MVP): что обязательно в первой версии, а что потом
— 3 риска и как проверить идею до того, как строить
```

## 2. Текст для лендинга

```
Ты — копирайтер посадочных страниц. Напиши структуру и тексты лендинга под мой продукт.

Продукт: [что это, для кого]
Боль клиента: [что болит у аудитории]
Оффер: [что предлагаю, цена, чем отличаюсь]

Дай по блокам:
1) Заголовок + подзаголовок (3 варианта)
2) Блок «узнаёшь себя?» — боли аудитории
3) Что получит клиент (выгоды, не функции)
4) Как это работает (3-4 шага)
5) Призыв к действию
Тон — [по моему вкусу]. Под рынок [страна/регион].
```

## 3. Сценарий простого бота

```
Ты — продуктовый аналитик. Опиши сценарий Telegram-бота под мою задачу — так, чтобы по нему можно было собрать MVP.

Задача бота: [что должен делать — заявки / FAQ / запись / прогрев]
Аудитория: [кто пользуется]
Что в итоге: [заявка мне / оплата / контакт]

Дай:
— Путь пользователя по шагам (от /start до цели)
— Какие кнопки/сообщения на каждом шаге
— Что бот спрашивает и что сохраняет
— Где подключить меня (живого) и на каком шаге
Без кода — только логика и тексты сообщений.
```

## 4. Проверка спроса до запуска

```
Ты — стартап-ментор. Помоги проверить спрос на продукт ДО того, как я его построю.

Идея: [что хочу запустить]
Аудитория: [кто клиент, где его искать]

Дай:
— 3 быстрых способа проверить спрос за неделю без готового продукта
— Какой вопрос задать потенциальным клиентам, чтобы понять, купят ли (без «вам было бы интересно?»)
— По какому сигналу пойму, что идею стоит строить, а какой — стоп
Конкретно и реалистично для соло-фаундера.
```
